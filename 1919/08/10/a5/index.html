1.神奇的口袋
描述
有n个物体，体积为a1,a2,...,an,给定一个数字K,希望选出一部分物品总体积为K(以40为例),问总共有多少种选法
解法一：递归
#include<iostream>
using namespace std;
int a[30];int N;
int Ways(int w,int k){
    if(w==0) return 1;
    if(k<=0) return 0;
    return Ways(w,k-1)+Ways(w-a[k],k-1);//无非是第k个选或者不选
}//代表从前K个中凑成w的方法数
int main(){
    cin>>N;
    for(int i=1;i<=N;i++){
        cin>>a[i];
    }
    cout<<Ways(N,40);
    return 0;
}
解法二：动规
#include<iostream>
using namespace std;
int main(){
    int a[30];
    int N;
    int Way[50][30];//Way[i][j]表示从前j个中选出和为i的种数
    cin>>N;
    memswt(Way,0,sizeof(Way));
    for(int i=1;i< =N;i++){
        cin>>a[i];
        Way[0][i]=1;//什么都不选属于一种方法
    }
    Way[0][0]=1;
    for(int i=1;i< =40;i++){
        for(int j=1;j< =N;j++){
            Way[i][j]=Way[i][j-1];
            if(i-a[j]> =0){
                Way[i][j]+=Way[i-a[j]][j-1];
            }
        }
    }
    cout<<Way[40][N];
    return 0;
}
2.0-1背包问题
题目描述
有N个物品和一个体积为M的背包，第i个物品的体积和价值分别为w[i],d[i]，求解将哪些物品放入背包内可以使得价值之和最大
N<=3500,M<=13000
思路：
首先仿造上一个题目，可以用一个二维数组来存放数据，F[i][j]表示从前i种物品中取出总体积不超过j的最优价值
考虑边界情况
如果 w[1]>j,F[1][j]=0;如果 w[1]<=j,F[1][j]=d[1];

递推式：F[i][j]=max(F[i-1][j],F[i-1][j-w[i]]+d[i]),前题当然是j比w[i]大
前面这两个题都要求你可以构造出这样一个数组Way[i][j],F[i][j]来存放数据
优化思路：
如果仅仅只有上面的思考，解决这样一道题是完全不够的，因为我们注意到，N<=3500,M<=13000
这就意味着我们需要开辟一个3500x13000大小的二维数组来存放数据，显然不可能
那么在某些情况下，可以使用滚动数组的方法来节省空间
本题就是一个例子：
在一开始的设想中，我们要通过第i-1行计算第i行
由递推式不难发现 F[i][j]=max(F[i-1][j],F[i-1][j-w[i]]+d[i])，F[i][j]只与它上一行的相同位置，和上一行靠左的某一位置的元素有关
它上一行右边是什么样的对它的值完全没有影响
那么我们完全可以将F[i][j+1]存放在F[i-1][j+1]中,这样做有一个前提，那就是F[i][j]右边的数先于F[i][j]被填上
确保右上方的数是完全“没用的”,所以要求我们从右往左循环，而不是从左往右
这样最后我们只需要用一个一维数组就可以了(当然可以这么做也与此题只需要最后一行的结果息息相关，如果中间的数据是我们需要的，就不应该随意替换)

3.分蛋糕
有一个矩形大蛋糕，它的宽和高为整数w,h,现在要把它切成m块小蛋糕，要求：每次只能将一块蛋糕切成两块，每块蛋糕的宽和高都为整数
要求计算出：最后m块小蛋糕中面积最大的那块蛋糕可能的最小面积
分析
有三个数据w,h,m，那么需要一个函数Way(w,h,m)来计算宽高为w,h,需要切m刀得到的一群蛋糕里面面积最大的可能具有的最小值
记SV为先横着切一刀的情况，SH为先竖着切一刀的情况
则way(w,h,m)=min(SV,SH)
SH:
竖着切一刀有w-1种切法
记Si为左边一块宽为i的情况，Si=min{max{way(i,h,k),way(w-i,h,m-1-k)},k=0,1,...,m-1}
Si有左右两块蛋糕，且最终需要合切m-1刀
SH=min{Si}
理解递推式
4.分解因数
描述
给出一个正整数a，要求分解成若干个正整数的乘积，即a = a1 * a2 * a3 * ... * an，并且1 < a1 <= a2 <= a3 <= ... <= an，问这样的分解的种数有多少。注意到a = a也是一种分解。
输入
第1行是测试数据的组数n，后面跟着n行输入。每组测试数据占1行，包括一个正整数a (1 < a < 32768)
输出
n行，每行输出对应一个输入。输出应是一个正整数，指明满足要求的分解的种数
样例输入
2
2
20
样例输出
1
4
其实思路有一些，但是如果对递归或者动态规划的一些思想不太熟悉的话，还是不一定好想的
下面是一种状态转移方程,用f(a,b)表示a分解为一系列不超过b的数
f(a,b)=f(a,b-1)+f(a/b,b);//前面一种是不分解出一个b的方案，后面一个是分解出一个b的方案
递归动规需要注意的是边界条件的刻画，b=1,return 0,a=1,return 1(表示这种情况完成了)
边界条件需要细心考量
代码示例
#include<iostream>
using namespace std;
int dp(int a,int b){
    if(a==1) return 1;
    if(b==1) return 0;
    if(a%b==0){
        return dp(a,b-1)+dp(a/b,b);
    }
    return dp(a,b-1);
}//积为a，最大分解为b的情况
int main(){
    int n;
    int N;
    cin>>n;
    while(n--){
        cin>>N;
        cout<<dp(N,N)<<endl;
    }
    return 0;
}
5.求最大公因数
记下辗转相除法的思想即可
#include<iostream>
using namespace std;
int dp(int a,int b){
    if(b==0) return a;
    int s=a%b;
    if(b==s) return b;
    else if(b>s){
        return dp(b,s);
    }
    else{
        return dp(s,b);
    }
}//保证a>b
int main(){
    int a;
    int b;
    cin>>a>>b;
    if(a>b){
        cout<<dp(a,b)<<endl;
    }
    else if(a<b){
        cout<<dp(b,a)<<endl;
    }
    else{
        cout<<a<<endl;
    }
    return 0;
}
求最大公约数可以使用辗转相除法：
假设a > b > 0，那么a和b的最大公约数等于b和a%b的最大公约数，然后把b和a%b作为新一轮的输入。
由于这个过程会一直递减，直到a%b等于0的时候，b的值就是所要求的最大公约数。
比如：
9和6的最大公约数等于6和9%6=3的最大公约数。
由于6%3==0，所以最大公约数为3。
6.完美覆盖

描述
一张普通的国际象棋棋盘，它被分成 8 乘 8 (8 行 8 列) 的 64 个方格。设有形状一样的多米诺牌，每张牌恰好覆盖棋盘上相邻的两个方格，即一张多米诺牌是一张 1 行 2 列或者 2 行 1 列的牌。那么，是否能够把 32 张多米诺牌摆放到棋盘上，使得任何两张多米诺牌均不重叠，每张多米诺牌覆盖两个方格，并且棋盘上所有的方格都被覆盖住？我们把这样一种排列称为棋盘被多米诺牌完美覆盖。这是一个简单的排列问题，同学们能够很快构造出许多不同的完美覆盖。但是，计算不同的完美覆盖的总数就不是一件容易的事情了。不过，同学们 发挥自己的聪明才智，还是有可能做到的。
现在我们通过计算机编程对 3 乘 n 棋盘的不同的完美覆盖的总数进行计算。

任务
对 3 乘 n 棋盘的不同的完美覆盖的总数进行计算。
输入
一次输入可能包含多行，每一行分别给出不同的 n 值 ( 即 3 乘 n 棋盘的列数 )。当输入 -1 的时候结束。

n 的值最大不超过 30.
输出
针对每一行的 n 值，输出 3 乘 n 棋盘的不同的完美覆盖的总数。
样例输入
2
8
12
-1
样例输出
3
153
2131
